OPA Bundle Source

In this sample OpaDotNet will use policy compiled as OPA policy bundle.

1. Create policy and data files

Create policy file quickstart/policy.rego with the following contents:

package example

default hello = false

hello {
    x := input.message
    x == data.world
}

Create data file quickstart/data.json with the following contents:

{
  "world": "world"
}

2. Compile policy bundle

In this example we will compile policy bundle manually, see Compilation section for more details.

opa build -t wasm -b -e example/hello ./quickstart

opa CLI will produce bundle.tar.gz file.

3. Add usings


using OpaDotNet.Wasm;

4. The code


// Create evaluator from compiled policy module.
using var engine = OpaEvaluatorFactory.CreateFromBundle(File.OpenRead("data/bundle.tar.gz"));

// External data is in the bundle already.

// Evaluate. Policy query will return false.
var deny = engine.EvaluatePredicate(new { message = "hi" });

if (deny.Result)
{
    // Should not get here.
}
else
{
    // Wrong password.
}

// Evaluate. Policy query will return true.
var approve = engine.EvaluatePredicate(new { message = "world" });

if (approve.Result)
{
    // Correct password.
}
else
{
    // Should not get here.
}
